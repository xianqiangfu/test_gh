# Elastic Net

## 什么是Elastic Net？

Elastic Net 是**结合 L1 和 L2 正则化的线性回归**。

简单说：**同时拥有岭回归和Lasso的优点。**

---

## 为什么需要Elastic Net？

### 岭回归的问题

- 不能做特征选择
- 所有特征都有权重

### Lasso的问题

- 特征相关时表现不稳定
- 当 p > n（特征数 > 样本数）时最多选 n 个特征

### Elastic Net的优势

- 既能特征选择（L1）
- 又能处理相关特征（L2）
- 结合两者的优点

---

## Elastic Net的数学

### 原始损失（均方误差）

```
L = Σ(yᵢ - (wxᵢ + b))² / n
```

### Elastic Net的损失

```
L = Σ(yᵢ - (wxᵢ + b))² / n + α₁ × Σ|w| + α₂ × Σw²
```

### 另一种表示

```
L = MSE + α × (λ × ||w||₁ + (1-λ) × ||w||² / 2)
```

- α：正则化强度
- λ：L1/L2 比例（0 ≤ λ ≤ 1）
  - λ = 0：纯岭回归
  - λ = 1：纯Lasso
  - 0 < λ < 1：Elastic Net

---

## 参数解释

### α：正则化强度

控制整体正则化的强度。

- α = 0：无正则化（普通线性回归）
- α 小：弱正则化
- α 大：强正则化

### λ：混合比例

控制 L1 和 L2 的比例。

- λ = 0：纯岭回归（只有 L2）
- λ = 1：纯Lasso（只有 L1）
- λ = 0.5：L1 和 L2 各占一半

---

## 直观理解

### 约束区域

- L1：菱形（可以做特征选择）
- L2：圆形（处理相关特征）
- Elastic Net：介于两者之间

```
L1 约束     Elastic Net     L2 约束
   ╱│╲          ╱│╲           ╱│╲
  ╱ │ ╲        ╱ │ ╲         ╱ │ ╲

╲   │   ╱    ╲   │   ╱     ╲   │   ╱

  ╲ │ ╱        ╲ │ ╱         ╲ │ ╱
   ╲│╱          ╲│╱           ╲│╱
```

### 效果

- L1 部分：让一些权重为0（特征选择）
- L2 部分：处理相关特征，保持稳定

---

## 何时使用Elastic Net？

### 适合的情况

✅ 特征数量很多
✅ 特征之间存在相关性
✅ 需要特征选择
✅ 样本数少于特征数（p > n）

### 不适合的情况

❌ 特征独立且都重要（用岭回归）
❌ 不需要特征选择（用岭回归）
❌ 特征不相关（用Lasso即可）

---

## Elastic Net vs 其他方法

| 特点 | 线性回归 | 岭回归 (L2) | Lasso (L1) | Elastic Net |
|------|---------|------------|-----------|-------------|
| 正则化 | 无 | L2 | L1 | L1 + L2 |
| 特征选择 | 无 | 无 | 有 | 有 |
| 处理相关 | - | 好 | 差 | 好 |
| p > n | - | 可以 | 最多 n 个 | 可以 |
| 稀疏解 | 无 | 无 | 是 | 是 |
| 可解释性 | 一般 | 一般 | 好 | 好 |

## 实际例子

### 例子：文本分类

有1000个词作为特征，其中一些词高度相关（如同义词）。

### Lasso

随机选择其中一个相关词，其他设为0，不稳定。

### 岭回归

所有词都有权重，但无法做特征选择。

### Elastic Net

- L1 部分：选择重要的词组
- L2 部分：相关的词都有权重，保持稳定

结果：既简洁又稳定。

## 超参数调优

### 调参策略

1. **网格搜索**

   ```
   α: 0.01, 0.1, 1, 10
   λ: 0, 0.25, 0.5, 0.75, 1
   ```

   共 5 × 4 = 20 种组合

2. **交叉验证**

   用验证集评估每种组合

3. **选择最佳组合**

   使验证误差最小的 (α, λ)

### 交叉验证曲线

```
验证误差
  ↑
  │  λ=0.5  ╱╲
  │        ╱  ╲
  │  λ=0.7 ╱    ╲
  │        ╱      ╲
  │  λ=0.3 ╱        ╲
  └─────────────────→ α
      最佳 α
```

为每个 λ 找最佳 α，再选整体最优。

---

## 参数选择的经验

### 默认值

通常从以下值开始尝试：

```
α: 0.1, 1, 10
λ: 0.5 (L1和L2各一半)
```

### 特定场景

| 场景 | α | λ |
|------|---|---|
| 需要强特征选择 | 中等 | 大（接近1） |
| 特征高度相关 | 中等 | 小（接近0） |
| p >> n | 大 | 0.5 左右 |
| 一般情况 | 1 | 0.5 |

---

## 数据预处理

### 标准化

像岭回归和Lasso一样，Elastic Net也需要标准化：

```
x' = (x - 均值) / 标准差
```

**原因**：
- 正则化约束权重大小
- 不同尺度的特征影响不同

---

## 缩减路径

### 权重变化

随着 α 增大，权重逐渐减小，一些为0：

```
α=0:   [5, 3, 2, 1, 0.5]
α=0.1: [4, 2.5, 1.5, 0.5, 0.2]
α=1:   [3, 2, 1, 0.3, 0]
α=10:  [1, 0.8, 0.5, 0, 0]
α=100: [0.2, 0.1, 0, 0, 0]
```

注意：相关特征的权重可能一起减小，不会像Lasso那样只有一个。

---

## 算法实现

### 坐标下降

类似Lasso，但现在有两个正则项。

### 近端梯度法

结合梯度下降和近端算子。

### 最小角回归（LARS）变体

适配Elastic Net。

---

## 实践技巧

### 1. 标准化

必须对特征标准化。

### 2. 交叉验证

用网格搜索找最佳 (α, λ)。

### 3. 从 λ=0.5 开始

L1和L2各半通常是好的起点。

### 4. 观察权重

观察哪些特征被选中，是否符合预期。

## 代码示例

### Scikit-learn

```python
from sklearn.linear_model import ElasticNet
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import GridSearchCV

# 标准化
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# 参数网格
param_grid = {
    'alpha': [0.01, 0.1, 1, 10],
    'l1_ratio': [0, 0.25, 0.5, 0.75, 1]  # l1_ratio 就是 λ
}

# Elastic Net
en = ElasticNet()

# 网格搜索
grid = GridSearchCV(en, param_grid, cv=5)
grid.fit(X_scaled, y)

# 最佳参数
print(grid.best_params_)

# 最佳模型
best_model = grid.best_estimator_

# 查看权重
print(best_model.coef_)
```

## 优势总结

### 相比岭回归

- 可以做特征选择
- 模型更简洁

### 相比Lasso

- 处理相关特征更好
- 当 p > n 时可以选择超过 n 个特征
- 更稳定

---

## 缺点

### 计算成本

需要调两个超参数，比单独调一个要麻烦。

### 解释性

比纯Lasso稍微复杂一些（L2部分）。

---

## 总结

**Elastic Net**：

1. 结合 L1 和 L2 正则化
2. 兼具特征选择和处理相关特征的能力
3. 适合高维数据和特征相关的情况
4. 需要调两个超参数（α, λ）
5. 综合性能通常最好

**核心思想**：
- L1 部分：稀疏性，特征选择
- L2 部分：稳定性，处理相关性
- 平衡两者，取长补短

**关键要点**：
- 标准化很重要
- 用网格搜索调参
- λ=0.5 是好的起点
- 综合性能通常最好

**记住**：Elastic Net就像"全能选手"，既有Lasso的特征选择能力，又有岭回归的稳定性！
