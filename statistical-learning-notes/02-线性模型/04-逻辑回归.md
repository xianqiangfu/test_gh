# 逻辑回归（分类问题）

## 什么是逻辑回归？

虽然名字里有"回归"，但逻辑回归其实是**分类算法**。

它的核心任务：**预测一个样本属于某一类别的概率**。

---

## 为什么叫"逻辑"？

因为使用了**逻辑函数 (Sigmoid函数)** 来进行变换。

这个函数把任意实数映射到 (0,1) 区间，正好表示概率。

---

## 二分类逻辑回归的详细推导

### 模型定义

给定输入特征 $\mathbf{x} = (x_1, x_2, ..., x_n)$，我们预测样本属于正类的概率：

$$P(y=1|\mathbf{x}) = \sigma(\mathbf{w}^T \mathbf{x} + b)$$

其中 $\sigma(z) = \frac{1}{1 + e^{-z}}$ 是Sigmoid函数。

### 为什么使用Sigmoid函数？

**直观理解**：我们需要一个函数，将线性回归的输出（任意实数）映射到[0,1]区间。

**数学推导**：

假设我们使用**对数几率（log-odds）**作为线性模型：

$$\log\left(\frac{p}{1-p}\right) = \mathbf{w}^T \mathbf{x} + b$$

其中：
- $p = P(y=1|\mathbf{x})$
- $\frac{p}{1-p}$ 称为几率（odds）
- $\log\left(\frac{p}{1-p}\right)$ 称为对数几率（log-odds）

从对数几率推导出Sigmoid函数：

$$\frac{p}{1-p} = e^{\mathbf{w}^T \mathbf{x} + b}$$

$$p = (1-p)e^{\mathbf{w}^T \mathbf{x} + b}$$

$$p = e^{\mathbf{w}^T \mathbf{x} + b} - pe^{\mathbf{w}^T \mathbf{x} + b}$$

$$p(1 + e^{\mathbf{w}^T \mathbf{x} + b}) = e^{\mathbf{w}^T \mathbf{x} + b}$$

$$p = \frac{e^{\mathbf{w}^T \mathbf{x} + b}}{1 + e^{\mathbf{w}^T \mathbf{x} + b}} = \frac{1}{1 + e^{-(\mathbf{w}^T \mathbf{x} + b)}}$$

这就是**Sigmoid函数**！

### 最大似然估计

#### 似然函数

对于n个独立样本 $\{(\mathbf{x}_i, y_i)\}_{i=1}^n$，其中 $y_i \in \{0, 1\}$。

单个样本的概率：

$$P(y_i|\mathbf{x}_i) = p_i^{y_i} (1-p_i)^{1-y_i}$$

其中 $p_i = \sigma(\mathbf{w}^T \mathbf{x}_i + b)$。

对数似然函数：

$$\ell(\mathbf{w}, b) = \sum_{i=1}^n [y_i \log(p_i) + (1-y_i) \log(1-p_i)]$$

#### 梯度推导

我们需要最大化对数似然，等价于最小化负对数似然（交叉熵损失）：

$$L(\mathbf{w}, b) = -\sum_{i=1}^n [y_i \log(p_i) + (1-y_i) \log(1-p_i)]$$

对单个样本的损失函数求导：

$$\frac{\partial L_i}{\partial p_i} = -\frac{y_i}{p_i} + \frac{1-y_i}{1-p_i}$$

利用链式法则：

$$\frac{\partial p_i}{\partial z_i} = \sigma(z_i)(1-\sigma(z_i)) = p_i(1-p_i)$$

其中 $z_i = \mathbf{w}^T \mathbf{x}_i + b$。

对权重的梯度：

$$\frac{\partial L_i}{\partial \mathbf{w}} = \frac{\partial L_i}{\partial p_i} \cdot \frac{\partial p_i}{\partial z_i} \cdot \frac{\partial z_i}{\partial \mathbf{w}}$$

$$= \left(-\frac{y_i}{p_i} + \frac{1-y_i}{1-p_i}\right) \cdot p_i(1-p_i) \cdot \mathbf{x}_i$$

$$= (p_i - y_i) \mathbf{x}_i$$

对偏置的梯度：

$$\frac{\partial L_i}{\partial b} = p_i - y_i$$

#### 梯度下降更新

$$\mathbf{w} \leftarrow \mathbf{w} - \eta \sum_{i=1}^n (p_i - y_i) \mathbf{x}_i$$

$$b \leftarrow b - \eta \sum_{i=1}^n (p_i - y_i)$$

其中 $\eta$ 是学习率。

### 决策边界

当预测概率等于0.5时：

$$\sigma(\mathbf{w}^T \mathbf{x} + b) = 0.5$$

$$\mathbf{w}^T \mathbf{x} + b = 0$$

这就是**决策边界**，在特征空间中是一个超平面。

**对于二维情况**：

$$w_1 x_1 + w_2 x_2 + b = 0$$

这是一条直线，将特征空间分为两个区域。

## 线性回归的问题

假设我们想预测邮件是否是垃圾邮件：

- 线性回归的输出可能是：2.5, -1.3, 0.7...
- 但概率应该在 0 到 1 之间

**问题**：线性回归的输出范围不对！而且使用均方误差会导致非凸的损失函数，难以优化。

---

## Sigmoid 函数

### 定义

```
σ(z) = 1 / (1 + e^(-z))
```

### 图像

```
σ(z)
1.0 │      ╭──────
    │     ╱
0.5 │    ╱
    │   ╱
0.0 │  ╱
    │ ╱
    └──────────→ z
```

### 性质

- 输入：任意实数
- 输出：0 到 1 之间
- z = 0 时，σ(z) = 0.5
- z → ∞ 时，σ(z) → 1
- z → -∞ 时，σ(z) → 0

### 导数

$$\sigma'(z) = \sigma(z)(1 - \sigma(z))$$

这个性质在梯度计算中非常重要！

---

## 逻辑回归的模型

### 完整公式

```
z = w₁x₁ + w₂x₂ + ... + wₙxₙ + b
p = σ(z) = 1 / (1 + e^(-z))
```

- 先计算线性组合 z
- 再通过 sigmoid 函数得到概率 p

### 分类规则

```
如果 p ≥ 0.5，预测为类别 1
如果 p < 0.5，预测为类别 0
```

---

## 直观理解

### 例子：预测学生是否通过考试

特征：
- 学习时间：x₁
- 作业完成率：x₂

模型：
```
z = 2 × 学习时间 + 1.5 × 作业完成率 - 3
p = σ(z)
```

如果学生：
- 学习了5小时
- 作业完成率80%

计算：
```
z = 2 × 5 + 1.5 × 0.8 - 3 = 8.2
p = 1 / (1 + e^(-8.2)) ≈ 0.9997
```

预测：几乎肯定会通过（概率99.97%）

---

## 决策边界

### 二分类情况

当 p = 0.5 时，z = 0

```
w₁x₁ + w₂x₂ + b = 0
```

这就是一条**决策边界**，把两类分开。

```
类别1区域
    │
    │   ●●●
    │  ●   ●
    │●     ●
━━━━┼━━━━━━━━━━ 决策边界
    │●     ●
    │  ●   ●
    │   ●●●
    │
  类别0区域
```

## 损失函数

### 交叉熵损失

逻辑回归使用**交叉熵损失**，而不是均方误差。

对于单个样本：

```
L = -[y × log(p) + (1-y) × log(1-p)]
```

- y = 1 时：L = -log(p)
- y = 0 时：L = -log(1-p)

### 直观理解

- 当真实标签是1，预测概率接近1时，损失小
- 当真实标签是1，预测概率接近0时，损失大

---

## 训练过程

### 步骤

1. **初始化参数**
   - 随机设置 w 和 b

2. **前向传播**
   - 计算 z 和 p

3. **计算损失**
   - 用交叉熵损失

4. **反向传播**
   - 计算梯度

5. **更新参数**
   - 用梯度下降

6. **重复**
   - 直到收敛

## 多分类逻辑回归（Softmax回归）

### 为什么需要多分类？

二分类逻辑回归只能处理两个类别，但现实问题往往有多个类别：
- 数字识别（0-9）
- 图像分类（猫、狗、鸟...）
- 情感分析（正面、负面、中性）

### 多分类方法对比

| 方法 | 策略 | 特点 |
|------|------|------|
| **One-vs-Rest** | 训练K个二分类器 | 每个分类器区分一类 vs 其他 |
| **Softmax回归** | 一次性训练K类 | 输出K个概率，更优雅 |

### Softmax回归模型

#### 模型定义

对于K个类别，我们为每个类别k定义一个权重向量 $\mathbf{w}_k$ 和偏置 $b_k$。

样本 $\mathbf{x}$ 属于第k类的概率：

$$P(y=k|\mathbf{x}) = \frac{e^{z_k}}{\sum_{j=1}^K e^{z_j}}$$

其中 $z_k = \mathbf{w}_k^T \mathbf{x} + b_k$。

#### Softmax函数的性质

1. **概率性质**：
   - 所有概率之和为1：$\sum_{k=1}^K P(y=k|\mathbf{x}) = 1$
   - 每个概率都在[0,1]之间

2. **不变性**：
   - 所有 $z_k$ 加上同一个常数，概率不变
   - 这意味着我们可以对 $z_k$ 进行数值稳定化

3. **数值稳定实现**：
   ```python
   # 稳定的Softmax实现
   z = z - np.max(z)  # 避免数值溢出
   exp_z = np.exp(z)
   return exp_z / np.sum(exp_z)
   ```

#### 损失函数

对于样本 $(\mathbf{x}, y)$，使用**交叉熵损失**：

$$L = -\log(P(y|\mathbf{x})) = -\log\left(\frac{e^{z_y}}{\sum_{j=1}^K e^{z_j}}\right)$$

简化后：

$$L = -z_y + \log\left(\sum_{j=1}^K e^{z_j}\right)$$

#### 梯度推导

对于类别 $k$：

$$\frac{\partial L}{\partial z_k} = \begin{cases} p_k - 1, & k = y \\ p_k, & k \neq y \end{cases}$$

可以统一写成：

$$\frac{\partial L}{\partial z_k} = p_k - \mathbb{I}(k=y)$$

其中 $\mathbb{I}$ 是指示函数。

#### 权重更新

$$\mathbf{w}_k \leftarrow \mathbf{w}_k - \eta (p_k - \mathbb{I}(k=y)) \mathbf{x}$$

$$b_k \leftarrow b_k - \eta (p_k - \mathbb{I}(k=y))$$

### One-vs-Rest策略

对于K个类别，训练K个二分类器：

1. **训练第k个分类器**：
   - 正类：第k类
   - 负类：其他所有类

2. **预测**：
   - 对于每个分类器，计算概率
   - 选择概率最高的类别

**缺点**：
- 需要训练K个模型
- 类别不平衡问题
- 可能出现预测矛盾

**Softmax的优势**：
- 一次训练，端到端优化
- 自然处理多类别
- 输出真正的概率分布

## 逻辑回归 vs 线性回归

| 特点 | 线性回归 | 逻辑回归 |
|------|---------|----------|
| 任务 | 回归 | 分类 |
| 输出范围 | (-∞, ∞) | (0, 1) |
| 激活函数 | 无 | Sigmoid |
| 损失函数 | 均方误差 | 交叉熵 |
| 可解释性 | 可解释 | 可解释 |

---

## 优点

✅ **简单高效**
- 训练快
- 计算量小

✅ **可解释性强**
- 可以看出每个特征的影响
- 权重表示特征的重要性

✅ **输出概率**
- 不仅给出分类，还给出概率
- 可以设定不同的阈值

✅ **理论成熟**
- 有完善的统计学基础
- 容易理解和解释

---

## 缺点

❌ **只能处理线性决策边界**
- 对于复杂的非线性问题表现差

❌ **对异常值敏感**
- 像线性回归一样

❌ **特征需要独立**
- 特征之间高度相关时会出问题

---

## 正则化详解

### 为什么需要正则化？

**过拟合问题**：
- 模型在训练集上表现很好
- 在测试集上表现很差
- 权重变得很大，对噪声敏感

**正则化的作用**：
- 限制权重的大小
- 防止过拟合
- 提高泛化能力

### L2 正则化（岭回归）

#### 损失函数

$$L = -\frac{1}{n}\sum_{i=1}^n [y_i \log(p_i) + (1-y_i) \log(1-p_i)] + \lambda \|\mathbf{w}\|_2^2$$

其中 $\|\mathbf{w}\|_2^2 = \sum_{j=1}^d w_j^2$ 是L2范数的平方。

#### 梯度

$$\frac{\partial L}{\partial \mathbf{w}} = \frac{1}{n}\sum_{i=1}^n (p_i - y_i)\mathbf{x}_i + 2\lambda \mathbf{w}$$

#### 特点

- **权重衰减**：让所有权重都变小，但不完全为0
- **平滑性**：倾向于选择权重分布均匀的解
- **数值稳定**：处处可导，优化容易
- **sklearn参数**：`C = 1/λ`，C越小，正则化越强

### L1 正则化（Lasso）

#### 损失函数

$$L = -\frac{1}{n}\sum_{i=1}^n [y_i \log(p_i) + (1-y_i) \log(1-p_i)] + \lambda \|\mathbf{w}\|_1$$

其中 $\|\mathbf{w}\|_1 = \sum_{j=1}^d |w_j|$ 是L1范数。

#### 特点

- **稀疏性**：让很多权重变为0
- **特征选择**：自动选择重要的特征
- **不可导**：在0点不可导，需要特殊优化方法（如坐标下降）
- **sklearn参数**：`penalty='l1'`

### Elastic Net

#### 损失函数

$$L = -\frac{1}{n}\sum_{i=1}^n [y_i \log(p_i) + (1-y_i) \log(1-p_i)] + \lambda_1 \|\mathbf{w}\|_1 + \lambda_2 \|\mathbf{w}\|_2^2$$

#### 特点

- 结合L1和L2的优点
- 既能做特征选择，又能保持模型的稳定性
- 适合特征相关性高的情况

### 正则化选择指南

| 情况 | 推荐正则化 |
|------|-----------|
| 特征很多，想选重要特征 | L1 |
| 特征相关性强 | L2 或 Elastic Net |
| 所有特征都可能重要 | L2 |
| 不确定，想试试 | Elastic Net |
| 默认选择 | L2 |

## 类别不平衡问题处理

### 什么是类别不平衡？

当不同类别的样本数量差异很大时，称为类别不平衡。

**例子**：
- 欺诈检测：1000个正常交易，只有10个欺诈交易
- 疾病诊断：999个健康样本，只有1个患病样本
- 罕见事件检测

### 类别不平衡的影响

#### 对准确率的影响

假设1000个样本中，990个正类，10个负类。

**模型A**：全预测为正类
- 准确率：990/1000 = 99%
- 但毫无意义！

**模型B**：真正识别出5个负类，误判5个正类
- 准确率：985/1000 = 98.5%
- 更有用！

#### 对决策边界的影响

- 决策边界会偏向少数类
- 少数类容易被误判
- 模型学习到的不是真实的边界

### 处理方法

#### 1. 数据层面方法

##### 过采样（Oversampling）

**随机过采样**：
- 简单复制少数类样本
- 容易导致过拟合

**SMOTE (Synthetic Minority Over-sampling Technique)**：
```python
from imblearn.over_sampling import SMOTE

# SMOTE过采样
smote = SMOTE(random_state=42)
X_resampled, y_resampled = smote.fit_resample(X, y)
```

**原理**：
1. 对每个少数类样本，找到k个最近邻
2. 在连线上随机生成新样本
3. 增加少数类样本的多样性

**优点**：
- 增加样本多样性
- 不是简单复制
- 缓解过拟合

**缺点**：
- 生成样本可能不合理
- 计算成本较高

##### 欠采样（Undersampling）

**随机欠采样**：
```python
from imblearn.under_sampling import RandomUnderSampler

# 欠采样
rus = RandomUnderSampler(random_state=42)
X_resampled, y_resampled = rus.fit_resample(X, y)
```

**原理**：
- 随机删除多数类样本
- 使类别平衡

**优点**：
- 计算效率高
- 减少多数类的影响

**缺点**：
- 丢失信息
- 可能删除重要样本

**Tomek Links**：
- 删除靠近决策边界的多数类样本
- 使类间边界更清晰

#### 2. 算法层面方法

##### 类别权重

**原理**：给少数类更高的权重，在损失函数中增加其重要性。

**sklearn实现**：
```python
from sklearn.linear_model import LogisticRegression

# 自动计算类别权重
model = LogisticRegression(class_weight='balanced')

# 手动设置权重
model = LogisticRegression(class_weight={0: 1, 1: 10})
```

**权重计算**（balanced模式）：
$$weight_k = \frac{n_{samples}}{n_{classes} \times n_k}$$

其中 $n_k$ 是第k类的样本数。

**效果**：
- 少数类的误判代价更高
- 决策边界向多数类移动

##### 阈值调整

**原理**：调整分类阈值，而不是默认的0.5。

```python
# 预测概率
y_prob = model.predict_proba(X_test)[:, 1]

# 调整阈值（默认是0.5）
threshold = 0.3  # 降低阈值，提高召回率
y_pred = (y_prob >= threshold).astype(int)
```

**阈值选择方法**：

1. **基于业务需求**：
   - 欺诈检测：高召回率，低阈值
   - 垃圾邮件：高精确率，高阈值

2. **基于ROC曲线**：
   - 选择FPR和TPR平衡的点

3. **基于F1分数**：
   - 最大化F1分数

```python
from sklearn.metrics import f1_score
import numpy as np

# 寻找最优阈值
thresholds = np.arange(0, 1, 0.01)
f1_scores = [f1_score(y_test, (y_prob >= t).astype(int)) for t in thresholds]
best_threshold = thresholds[np.argmax(f1_scores)]
```

##### 代价敏感学习

**原理**：不同类型的错误有不同的代价。

**代价矩阵**：

```
                预测
            正类    负类
实际 正类   0      C_FN
    负类   C_FP   0
```

其中：
- C_FN：漏报代价（FN的代价）
- C_FP：误报代价（FP的代价）

**sklearn实现**：
```python
# 使用sample_weight指定每个样本的权重
sample_weights = np.where(y_train == 1, 10, 1)  # 正类权重10倍
model.fit(X_train, y_train, sample_weight=sample_weights)
```

#### 3. 集成方法

##### Balanced Random Forest

```python
from imblearn.ensemble import BalancedRandomForestClassifier

model = BalancedRandomForestClassifier(random_state=42)
model.fit(X_train, y_train)
```

##### Easy Ensemble

- 对多数类进行多次欠采样
- 训练多个分类器
- 集成投票

```python
from imblearn.ensemble import EasyEnsembleClassifier

model = EasyEnsembleClassifier(random_state=42)
model.fit(X_train, y_train)
```

### 评估指标的选择

类别不平衡时，不要只用准确率！

**推荐指标**：
- **F1分数**：平衡精确率和召回率
- **AUC**：不依赖阈值
- **PR曲线**：关注正类的性能
- **召回率**：如果不允许漏报
- **精确率**：如果不允许误报

### 处理策略选择

| 场景 | 推荐方法 |
|------|----------|
| 数据量大 | 欠采样 |
| 数据量小 | 过采样（SMOTE） |
| 想保留所有信息 | 类别权重 |
| 计算资源有限 | 类别权重 |
| 需要高召回率 | 降低阈值 + SMOTE |
| 需要高精确率 | 提高阈值 + 类别权重 |

## 模型评估指标（分类问题）

### 混淆矩阵

```
                预测
            正类    负类
实际 正类   TP      FN
    负类   FP      TN
```

- **TP**：True Positive（真阳性）- 正确预测为正类
- **TN**：True Negative（真阴性）- 正确预测为负类
- **FP**：False Positive（假阳性）- 错误预测为正类
- **FN**：False Negative（假阴性）- 错误预测为负类

### 核心指标

#### 准确率（Accuracy）

$$Accuracy = \frac{TP + TN}{TP + TN + FP + FN}$$

**含义**：所有预测中正确的比例

**问题**：类别不平衡时误导

#### 精确率（Precision）

$$Precision = \frac{TP}{TP + FP}$$

**含义**：预测为正类的样本中，真正是正类的比例

**关注**：控制误报（FP）

#### 召回率（Recall / TPR）

$$Recall = \frac{TP}{TP + FN}$$

**含义**：实际正类中，被正确识别的比例

**关注**：控制漏报（FN）

#### F1分数

$$F1 = 2 \times \frac{Precision \times Recall}{Precision + Recall}$$

**含义**：精确率和召回率的调和平均

**特点**：平衡精确率和召回率

#### 特异性（Specificity / TNR）

$$Specificity = \frac{TN}{TN + FP}$$

**含义**：实际负类中，被正确识别的比例

### ROC和AUC

#### ROC曲线

ROC（Receiver Operating Characteristic）曲线显示不同阈值下的TPR和FPR：

```
TPR
  │    ●
  │   ╱
  │  ╱
  │ ╱
  │╱
  └─────────→ FPR
```

#### AUC

AUC（Area Under Curve）是ROC曲线下的面积：

- AUC = 1.0：完美分类器
- AUC = 0.5：随机猜测
- AUC < 0.5：比随机还差

**优势**：
- 不依赖分类阈值
- 适合类别不平衡的情况
- 可以比较不同模型

### 多分类评估

对于多分类问题：

1. **宏平均（Macro Average）**：
   - 对每个类别计算指标，再平均
   - 平等对待所有类别

2. **微观平均（Micro Average）**：
   - 先计算TP, FP, FN的总和，再计算指标
   - 受大类影响大

3. **加权平均（Weighted Average）**：
   - 按类别样本数加权平均

```python
from sklearn.linear_model import LogisticRegression
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
from sklearn.model_selection import cross_val_score

# 生成数据
X, y = make_classification(

    random_state=42
)

penalties = [
    ('None', None, 1.0),
    ('L2 (C=1)', 'l2', 1.0),
    ('L2 (C=0.1)', 'l2', 0.1),
    ('L2 (C=10)', 'l2', 10.0),
    ('L1 (C=1)', 'l1', 1.0),
    ('Elastic Net (l1_ratio=0.5)', 'elasticnet', 1.0),

print("正则化方法对比:")
print("=" * 60)
print(f"{'方法':<25} {'CV准确率':<15} {'非零权重数':<15}")
print("-" * 60)

for name, penalty, C in penalties:
    if penalty is None:
        # 不使用正则化（需要使用liblinear solver）
        model = LogisticRegression(
            penalty='l2',
            C=1e10,  # 极大的C相当于无正则化
            solver='liblinear',
            random_state=42
        )
    elif penalty == 'elasticnet':
        model = LogisticRegression(
            penalty='elasticnet',
            C=C,
            l1_ratio=0.5,
            solver='saga',
            max_iter=1000,
            random_state=42
        )
    else:
        model = LogisticRegression(
            penalty=penalty,
            C=C,
            solver='liblinear',
            random_state=42
        )
    
    # 交叉验证
    cv_scores = cross_val_score(model, X_scaled, y, cv=5)

    # 统计非零权重
    non_zero_weights = np.sum(np.any(model.coef_ != 0, axis=0))

for name, penalty, C in [('L2', 'l2', 1.0), ('L1', 'l1', 1.0), ('Elastic Net', 'elasticnet', 1.0)]:
    if penalty == 'elasticnet':
        model = LogisticRegression(penalty=penalty, C=C, l1_ratio=0.5, solver='saga', max_iter=1000, random_state=42)
    else:
        model = LogisticRegression(penalty=penalty, C=C, solver='liblinear', random_state=42)
    
    model.fit(X_scaled, y)
```

### 5. 阈值优化

```python
from sklearn.metrics import precision_recall_curve, roc_curve

# 使用不平衡数据
X, y = make_classification(
    n_samples=1000,
    n_features=20,
    n_informative=10,
    weights=[0.9, 0.1],
    random_state=42
)

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42, stratify=y)

# 训练模型
model = LogisticRegression(class_weight='balanced', random_state=42)
model.fit(X_train, y_train)
y_prob = model.predict_proba(X_test)[:, 1]

# 方法1：基于F1分数优化
precisions, recalls, thresholds = precision_recall_curve(y_test, y_prob)
f1_scores = 2 * (precisions * recalls) / (precisions + recalls + 1e-8)
best_threshold_f1 = thresholds[np.argmax(f1_scores[:-1])]  # 最后一个threshold被忽略

print(f"基于F1分数的最优阈值: {best_threshold_f1:.3f}")
print(f"对应F1分数: {f1_scores[np.argmax(f1_scores[:-1])]:.3f}")

# 方法2：基于Youden's J统计量（最大化TPR - FPR）
fpr, tpr, thresholds_roc = roc_curve(y_test, y_prob)
youden_j = tpr - fpr
best_threshold_youden = thresholds_roc[np.argmax(youden_j)]

print(f"\n基于Youden's J的最优阈值: {best_threshold_youden:.3f}")
print(f"对应Youden's J: {youden_j[np.argmax(youden_j)]:.3f}")

# 方法3：基于业务需求（例如，要求召回率至少90%）
desired_recall = 0.9
threshold_recall = thresholds[np.where(recalls >= desired_recall)[0][0]]
print(f"\n满足{desired_recall*100}%召回率的阈值: {threshold_recall:.3f}")

# 对比不同阈值
print("\n不同阈值的效果:")
print("=" * 70)
print(f"{'阈值':<10} {'准确率':<12} {'精确率':<12} {'召回率':<12} {'F1分数':<12}")
print("-" * 70)

thresholds_to_test = [0.1, 0.3, 0.5, best_threshold_f1, best_threshold_youden]
for threshold in thresholds_to_test:
    y_pred_t = (y_prob >= threshold).astype(int)
    acc = accuracy_score(y_test, y_pred_t)
    prec = precision_score(y_test, y_pred_t, zero_division=0)
    rec = recall_score(y_test, y_pred_t, zero_division=0)
    f1 = f1_score(y_test, y_pred_t, zero_division=0)
    print(f"{threshold:<10.3f} {acc:<12.3f} {prec:<12.3f} {rec:<12.3f} {f1:<12.3f}")

# 绘制阈值变化曲线
import matplotlib.pyplot as plt

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

# Precision-Recall曲线
axes[0].plot(recalls[:-1], precisions[:-1], 'b-', alpha=0.7)
axes[0].set_xlabel('Recall')
axes[0].set_ylabel('Precision')
axes[0].set_title('Precision-Recall Curve')
axes[0].grid(True, alpha=0.3)

# 标记最优F1点
best_idx = np.argmax(f1_scores[:-1])
axes[0].plot(recalls[best_idx], precisions[best_idx], 'ro', markersize=10)
axes[0].annotate(f'Best F1\nThreshold={best_threshold_f1:.3f}', 
                 xy=(recalls[best_idx], precisions[best_idx]),
                 xytext=(recalls[best_idx]-0.3, precisions[best_idx]-0.1),
                 arrowprops=dict(arrowstyle='->'))

# ROC曲线
axes[1].plot(fpr, tpr, 'b-', alpha=0.7)
axes[1].plot([0, 1], [0, 1], 'r--', alpha=0.5)
axes[1].set_xlabel('False Positive Rate')
axes[1].set_ylabel('True Positive Rate')
axes[1].set_title('ROC Curve')
axes[1].grid(True, alpha=0.3)

# 标记最优Youden's J点
best_idx_youden = np.argmax(youden_j)
axes[1].plot(fpr[best_idx_youden], tpr[best_idx_youden], 'ro', markersize=10)
axes[1].annotate(f'Best Youden\nThreshold={best_threshold_youden:.3f}', 
                 xy=(fpr[best_idx_youden], tpr[best_idx_youden]),
                 xytext=(fpr[best_idx_youden]+0.2, tpr[best_idx_youden]-0.2),
                 arrowprops=dict(arrowstyle='->'))

plt.tight_layout()
plt.savefig('threshold_optimization.png', dpi=150, bbox_inches='tight')
print("\n图表已保存为 'threshold_optimization.png'")
```

```python
# 如果没有数据，这里使用模拟数据演示完整流程
np.random.seed(42)
n_samples = 1000

# 生成模拟特征
data = {

    'sex': np.random.binomial(1, 0.6, n_samples),
    'cp': np.random.randint(0, 4, n_samples),  # 胸痛类型
    'trestbps': np.random.normal(130, 18, n_samples),  # 静息血压
    'chol': np.random.normal(245, 50, n_samples),  # 胆固醇
    'fbs': np.random.binomial(1, 0.15, n_samples),  # 空腹血糖
    'restecg': np.random.randint(0, 3, n_samples),  # 静息心电图
    'thalach': np.random.normal(150, 20, n_samples),  # 最大心率
    'exang': np.random.binomial(1, 0.33, n_samples),  # 运动诱发心绞痛
    'oldpeak': np.random.exponential(1, n_samples),  # ST段压低
    'slope': np.random.randint(0, 3, n_samples),  # ST段斜率

    'thal': np.random.randint(0, 4, n_samples),  # 地中海贫血

risk_score = (
    (data['age'] - 50) * 0.02 +
    data['cp'] * 0.3 +
    data['exang'] * 0.5 +
    data['oldpeak'] * 0.4 +
    data['ca'] * 0.6 +
    np.random.normal(0, 0.5, n_samples)
)
data['target'] = (risk_score > 0.5).astype(int)

df = pd.DataFrame(data)

print(f"\n训练集大小: {X_train.shape[0]}")
print(f"测试集大小: {X_test.shape[0]}")
```

### 7. 从零实现逻辑回归

```python
import numpy as np

class LogisticRegressionFromScratch:
    """从零实现的逻辑回归"""
    
    def __init__(self, learning_rate=0.01, n_iterations=1000, lambda_=0):
        """
        参数:
        - learning_rate: 学习率
        - n_iterations: 迭代次数
        - lambda_: L2正则化系数
        """
        self.learning_rate = learning_rate
        self.n_iterations = n_iterations
        self.lambda_ = lambda_
        self.weights = None
        self.bias = None
        self.losses = []
    
    def _sigmoid(self, z):
        """Sigmoid函数"""
        return 1 / (1 + np.exp(-np.clip(z, -250, 250)))  # 数值稳定
    
    def _initialize_parameters(self, n_features):
        """初始化参数"""
        self.weights = np.zeros(n_features)
        self.bias = 0
    
    def _compute_loss(self, X, y):
        """计算交叉熵损失"""
        n_samples = X.shape[0]
        
        # 前向传播
        z = np.dot(X, self.weights) + self.bias
        y_pred = self._sigmoid(z)
        
        # 交叉熵损失
        epsilon = 1e-15  # 避免log(0)
        loss = -np.mean(y * np.log(y_pred + epsilon) + 
                       (1 - y) * np.log(1 - y_pred + epsilon))
        
        # L2正则化
        reg_loss = (self.lambda_ / (2 * n_samples)) * np.sum(self.weights ** 2)
        
        return loss + reg_loss
    
    def fit(self, X, y):
        """训练模型"""
        n_samples, n_features = X.shape
        
        # 初始化参数
        self._initialize_parameters(n_features)
        
        # 梯度下降
        for i in range(self.n_iterations):
            # 前向传播
            z = np.dot(X, self.weights) + self.bias
            y_pred = self._sigmoid(z)
            
            # 计算梯度
            dw = (1 / n_samples) * np.dot(X.T, (y_pred - y)) + \
                 (self.lambda_ / n_samples) * self.weights
            db = (1 / n_samples) * np.sum(y_pred - y)
            
            # 更新参数
            self.weights -= self.learning_rate * dw
            self.bias -= self.learning_rate * db
            
            # 记录损失
            if i % 100 == 0:
                loss = self._compute_loss(X, y)
                self.losses.append(loss)
                if i % 1000 == 0:
                    print(f"Iteration {i}, Loss: {loss:.4f}")
    
    def predict_proba(self, X):
        """预测概率"""
        z = np.dot(X, self.weights) + self.bias
        return self._sigmoid(z)
    
    def predict(self, X, threshold=0.5):
        """预测类别"""
        y_pred_proba = self.predict_proba(X)
        return (y_pred_proba >= threshold).astype(int)
    
    def get_params(self):
        """获取模型参数"""
        return {
            'weights': self.weights,
            'bias': self.bias
        }

# 测试
if __name__ == "__main__":
```

## 常见面试题

### Q: 为什么逻辑回归不用均方误差？

**A**: 
1. **非凸性**：均方误差配合sigmoid函数会导致损失函数非凸，有多个局部最优
2. **梯度消失**：sigmoid函数在两端梯度很小，导致梯度下降很慢
3. **理论依据**：交叉熵是最大似然估计的直接结果，有统计学基础

### Q: 逻辑回归和线性模型的关系？

**A**: 
- 逻辑回归是在线性模型基础上加了sigmoid函数
- 对数几率是线性的：log(p/(1-p)) = w^T x + b
- 这使得逻辑回归保持了线性模型的可解释性

### Q: 逻辑回归怎么处理多分类？

**A**: 
1. **One-vs-Rest (OvR)**：训练K个二分类器，每个区分一类vs其他
2. **Softmax回归**：直接多分类，输出K个概率分布
3. **One-vs-One (OvO)**：训练K(K-1)/2个二分类器，每对类别一个

**推荐**：Softmax回归，因为它更优雅且端到端优化。

### Q: 逻辑回归的假设是什么？

**A**:
1. **线性关系**：对数几率与特征线性相关
2. **独立性**：样本之间相互独立
3. **无多重共线性**：特征之间不应高度相关
4. **大样本**：需要足够的数据来估计参数
5. **无完全分离**：不存在完全可分的类别（否则权重会趋于无穷大）

### Q: 什么时候不使用逻辑回归？

**A**:
1. **非线性决策边界**：当类别需要非线性边界时
2. **复杂交互**：当特征间有复杂交互关系时
3. **高维稀疏数据**：文本分类等场景可能用线性SVM更好
4. **需要概率校准**：当概率需要精确校准时

### Q: 如何处理类别不平衡？

**A**:
1. **数据层面**：过采样（SMOTE）、欠采样
2. **算法层面**：类别权重、代价敏感学习
3. **评估层面**：使用F1、AUC、PR曲线等指标
4. **决策层面**：调整分类阈值

### Q: L1和L2正则化的区别？

**A**:
- **L2**：权重衰减，让权重变小但不为0，适合一般情况
- **L1**：产生稀疏解，让很多权重为0，适合特征选择
- **Elastic Net**：结合两者，适合特征相关性高的情况

### Q: 为什么需要特征缩放？

**A**:
1. **梯度下降**：不同尺度的特征导致梯度不稳定
2. **收敛速度**：缩放后收敛更快
3. **正则化**：L1/L2对尺度敏感，需要统一尺度
4. **Sigmoid函数**：特征值过大会导致sigmoid饱和

### Q: 逻辑回归如何处理特征交互？

**A**:
1. **手动添加**：创建交互项，如x1 * x2
2. **多项式特征**：使用PolynomialFeatures
3. **核方法**：逻辑回归本身不支持，但可以用SVM

### Q: 概率校准是什么？什么时候需要？

**A**:
- **含义**：让模型输出的概率更准确反映真实概率
- **方法**：Platt scaling（sigmoid校准）、Isotonic regression
- **何时需要**：
  - 需要准确概率的决策（如风险评估）
  - 类别不平衡严重
  - 不同模型比较概率输出

## 总结

**逻辑回归**：

1. **核心思想**：使用sigmoid函数将线性输出映射到概率
2. **训练方法**：最大似然估计，交叉熵损失
3. **优势**：简单、快速、可解释、输出概率
4. **局限**：只能处理线性决策边界，对特征独立敏感

**多分类**：
- Softmax回归是自然的多分类扩展
- 输出真正的概率分布
- 比OvR方法更优雅

**类别不平衡**：
- 数据层面：过采样、欠采样
- 算法层面：类别权重、代价敏感学习
- 评估层面：F1、AUC、PR曲线

**正则化**：
- L2：防止过拟合，默认选择
- L1：特征选择，稀疏性
- Elastic Net：结合两者优点

**实践建议**：
1. 总是进行特征缩放
2. 使用交叉验证选择超参数
3. 对于类别不平衡，使用类别权重
4. 关注F1和AUC，不只是准确率
5. 尝试不同的正则化方法
6. 分析特征重要性，验证可解释性

**核心要点**：
- sigmoid把线性输出映射到概率
- 交叉熵是合适的损失函数
- 可以设置阈值调整分类策略
- 正则化防止过拟合
- Softmax扩展到多分类

逻辑回归是机器学习的"瑞士军刀"——简单但强大，在很多场景下都能用！它不仅是一个实用的分类器，更是理解更复杂模型的基础。
