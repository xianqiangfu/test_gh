# 交叉验证

## 什么是交叉验证？

交叉验证是**评估模型性能的标准方法**。

简单说：**把数据分成训练集和测试集，用训练集训练模型，用测试集评估模型。**

---

## 为什么需要交叉验证？

### 单次划分的问题

如果只用一次划分：

- 评估结果可能偶然
- 无法充分利用数据
- 难以了解模型稳定性

### 交叉验证的优势

✅ **更可靠的评估**
- 多次评估，结果更稳定

✅ **充分利用数据**
- 每个样本都被用到

✅ **了解方差**
- 可以看到模型的波动

---

## K折交叉验证

### 什么是K折？

把数据分成 K 份：

```
所有数据
    │
    ├─ 1折
    ├─ 2折
    ├─ ...
    └─ K折
```

### 过程

轮流用每一折做测试集，其余做训练集：

```
第1轮: [测试] [训练] [训练] [训练] [训练]
第2轮: [训练] [测试] [训练] [训练] [训练]
第3轮: [训练] [训练] [测试] [训练] [训练]
...
```

### 示例（5折）

```
第1轮:
  训练: [2][3][4][5]
  测试: [1]

第2轮:
  训练: [1][3][4][5]
  测试: [2]

第3轮:
  训练: [1][2][4][5]
  测试: [3]

第4轮:
  训练: [1][2][3][5]
  测试: [4]

第5轮:
  训练: [1][2][3][4]
  测试: [5]
```

---

## K的选择

### 常用K值

| K | 特点 | 适用 |
|---|------|------|
| 3 | 快但不太稳定 | 快速实验 |
| 5 | 平衡 | 最常用 |
| 10 | 更稳定 | 标准选择 |
| n | 最稳定但慢 | 小数据 |

### 如何选择？

- **数据量大**：K=5
- **数据量小**：K=10
- **小数据**：LOOCV
- **快速实验**：K=3

---

## 交叉验证的输出

### 各折的结果

```
第1折: 0.85
第2折: 0.82
第3折: 0.87
第4折: 0.84
第5折: 0.86
```

### 总结指标

**平均值**：整体性能

```
平均准确率 = (0.85 + 0.82 + 0.87 + 0.84 + 0.86) / 5 = 0.848
```

**标准差**：稳定性

```
标准差 = 0.018
```

标准差小 → 稳定
标准差大 → 不稳定

## 留一法

### 什么是LOOCV？

每次留一个样本做测试，其余做训练。

### 过程

```
第1次: 测试[1], 训练[2..n]
第2次: 测试[2], 训练[1,3..n]
...
第n次: 测试[n], 训练[1..n-1]
```

### 特点

**优点**：
- 评估最准确
- 充分利用数据

**缺点**：
- 计算量大（n次训练）
- 小数据时可能高估

---

## 分层K折

### 什么是分层？

保持各类别的比例：

```
原始数据: 60% A类, 40% B类

每折:
  折1: 60% A, 40% B
  折2: 60% A, 40% B
  ...
```

### 什么时候用？

**类别不平衡**时：

- 类别1: 100样本
- 类别2: 1000样本

普通K折可能某些折类别1很少。

---

## 代码示例

### K折交叉验证

```python
from sklearn.model_selection import cross_val_score, KFold
from sklearn.ensemble import RandomForestClassifier

# 创建模型
model = RandomForestClassifier()

# K折交叉验证
scores = cross_val_score(model, X, y, cv=5)

print(f"各折分数: {scores}")
print(f"平均分数: {scores.mean():.3f}")
print(f"标准差: {scores.std():.3f}")
```

### 自定义K折

```python
from sklearn.model_selection import KFold

# 创建K折
kf = KFold(n_splits=5, shuffle=True, random_state=42)

# 交叉验证
scores = cross_val_score(model, X, y, cv=kf)
```

### 分层K折

```python
from sklearn.model_selection import StratifiedKFold

# 创建分层K折
skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

# 交叉验证
scores = cross_val_score(model, X, y, cv=skf)
```

## 交叉验证 vs 简单划分

| 特点 | 简单划分 | 交叉验证 |
|------|---------|----------|
| 评估次数 | 1次 | K次 |
| 利用数据 | 部分 | 全部 |
| 稳定性 | 低 | 高 |
| 计算成本 | 低 | 高K倍 |

## 应用场景

| 场景 | 推荐 |
|------|------|
| 模型选择 | 交叉验证 |
| 参数调优 | 交叉验证 |
| 性能评估 | 交叉验证 |
| 快速实验 | 简单划分 |

## 常见问题

### Q: 交叉验证会过拟合吗？

A:
- 交叉验证本身不会
- 但如果多次调参，可能会
- 需要独立的测试集

### Q: K越大越好吗？

A:
- 不一定
- K大：稳定但慢
- K小：快但可能不稳定
- 通常K=5或10

### Q: 什么时候用LOOCV？

A:
- 数据很少（<100样本）
- 需要最准确评估

---

## 总结

**交叉验证**：

1. 把数据分成K份
2. 轮流用每一份测试
3. 计算平均性能和标准差
4. K=5或10最常用
5. 更可靠、更充分利用数据

**核心思想**：
- 多次评估
- 充分利用数据
- 更稳定的评估

**关键要点**：
- K=5最常用
- 数据少用LOOCV
- 类别不平衡用分层
- 看平均值和标准差

**记住**：交叉验证就像"多次考试"，考多次比考一次更准确！
