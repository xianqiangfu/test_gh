# 评估指标（分类）

## 什么是评估指标？

评估指标是**衡量分类模型好坏的标准**。

简单说：**用数值告诉我们模型表现如何。**

---

## 混淆矩阵

### 什么是混淆矩阵？

混淆矩阵是**分类结果的表格**：

```
                预测
            正类    负类
实际 正类   TP      FN
    负类   FP      TN
```

### 各项含义

- **TP**：True Positive（真阳性）
  - 实际正类，预测正类
  - 预测正确

- **FN**：False Negative（假阴性）
  - 实际正类，预测负类
  - 预测错误

- **FP**：False Positive（假阳性）
  - 实际负类，预测正类
  - 预测错误

- **TN**：True Negative（真阴性）
  - 实际负类，预测负类
  - 预测正确

### 示例

预测100封邮件是否是垃圾邮件：

```
                预测
            垃圾   正常
实际 垃圾   80     5
    正常   3     12

TP=80, FN=5, FP=3, TN=12
```

---

## 准确率

### 定义

```
准确率 = (TP + TN) / (TP + TN + FP + FN)
```

即：正确预测的比例

### 示例

```
准确率 = (80 + 12) / 100 = 92%
```

### 问题

当类别不平衡时，准确率可能误导：

```
1000样本中，990正常，10垃圾

如果把所有都预测为正常：
  准确率 = 990/1000 = 99%
```

但这毫无意义！

---

## 精确率

### 定义

```
精确率 = TP / (TP + FP)
```

即：预测为正类的样本中，真正是正类的比例

### 含义

预测为正类的**可靠性**

### 示例

```
精确率 = 80 / (80 + 3) = 96.4%
```

预测为垃圾邮件的邮件中，96.4%真的是垃圾邮件。

---

## 召回率

### 定义

```
召回率 = TP / (TP + FN)
```

即：实际正类的样本中，被预测为正类的比例

### 含义

找到正类的**能力**

### 示例

```
召回率 = 80 / (80 + 5) = 94.1%
```

实际垃圾邮件中，94.1%被识别出来。

---

## F1分数

### 定义

```
F1 = 2 × (精确率 × 召回率) / (精确率 + 召回率)
```

即：精确率和召回率的调和平均

### 特点

- 平衡精确率和召回率
- 范围在[0,1]之间
- 越大越好

### 示例

```
F1 = 2 × (0.964 × 0.941) / (0.964 + 0.941)
   = 0.952
```

---

## 精确率 vs 召回率

### 权衡

通常精确率和召回率是矛盾的：

- 精确率高 → 召回率可能低
- 召回率高 → 精确率可能低

### 阈值调整

改变分类阈值可以调整：

```
阈值高 → 预测正类少 → 精确率高，召回率低
阈值低 → 预测正类多 → 精确率低，召回率高
```

### 何时优先哪个？

| 场景 | 优先 |
|------|------|
| 垃圾邮件检测 | 精确率 |
| 疾病筛查 | 召回率 |
| 搜索结果 | 精确率 |
| 故障检测 | 召回率 |

---

## ROC曲线

### 什么是ROC？

ROC（Receiver Operating Characteristic）曲线是**不同阈值下的TPR和FPR关系图**。

### TPR和FPR

```
TPR（真正率）= 召回率 = TP / (TP + FN)
FPR（假正率）= FP / (FP + TN)
```

### ROC曲线

```
TPR
  │    ●
  │   ╱
  │  ╱
  │ ╱
  │╱
  └─────────→ FPR
```

### 理想情况

```
TPR
  │    ╭───
  │    │   ╲
  │    │    ╲
  │    │     ╲
  │    │      ╲
  └───────────→ FPR
```

---

## AUC

### 什么是AUC？

AUC（Area Under Curve）是**ROC曲线下的面积**。

### 范围

- AUC = 1：完美分类器
- AUC = 0.5：随机猜测
- AUC < 0.5：比随机还差

### 优势

- 不依赖阈值
- 平衡各类别
- 适合比较模型

---

## 代码示例

### 混淆矩阵

```python
from sklearn.metrics import confusion_matrix
from sklearn.metrics import accuracy_score, precision_score, recall_score, f1_score

# 预测
y_pred = model.predict(X_test)

# 混淆矩阵
cm = confusion_matrix(y_test, y_pred)
print(cm)

# 指标
print(f"准确率: {accuracy_score(y_test, y_pred):.3f}")
print(f"精确率: {precision_score(y_test, y_pred):.3f}")
print(f"召回率: {recall_score(y_test, y_pred):.3f}")
print(f"F1分数: {f1_score(y_test, y_pred):.3f}")
```

### ROC和AUC

```python
from sklearn.metrics import roc_curve, auc, roc_auc_score

# 预测概率
y_prob = model.predict_proba(X_test)[:, 1]

# ROC曲线
fpr, tpr, thresholds = roc_curve(y_test, y_prob)
roc_auc = auc(fpr, tpr)

# 或直接计算AUC
roc_auc = roc_auc_score(y_test, y_prob)

print(f"AUC: {roc_auc:.3f}")
```

## 指标选择

| 场景 | 主要指标 |
|------|----------|
| 一般分类 | 准确率 |
| 类别不平衡 | 精确率、召回率、F1 |
| 搜索排序 | 精确率@k |
| 医疗诊断 | 召回率 |
| 广告点击 | AUC |

## 多分类

### 方法

1. **宏观平均**
   - 对每个类别计算指标，再平均

2. **微观平均**
   - 先计算TP, FP, FN的总和，再计算指标

3. **加权平均**
   - 按类别样本数加权平均

### 代码

```python
# 宏观平均
precision_macro = precision_score(y_test, y_pred, average='macro')

# 微观平均
precision_micro = precision_score(y_test, y_pred, average='micro')

# 加权平均
precision_weighted = precision_score(y_test, y_pred, average='weighted')
```

## 常见问题

### Q: 准确率什么时候不能用？

A:
- 类别不平衡时
- 某类更重要时
- 需要关注误判时

### Q: F1和准确率哪个好？

A:
- F1更适合类别不平衡
- 准确率适合平衡数据
- 看具体需求

### Q: AUC越大越好吗？

A:
- 是的
- AUC=1是完美
- AUC=0.5是随机

---

## 总结

**分类评估指标**：

1. 混淆矩阵：TP, FP, TN, FN
2. 准确率：正确预测比例
3. 精确率：预测正类的可靠性
4. 召回率：找到正类的能力
5. F1：平衡精确率和召回率
6. ROC/AUC：不依赖阈值的评估

**核心思想**：
- 不同指标关注不同方面
- 根据任务选择合适指标
- 类别不平衡时用F1

**关键要点**：
- 准确率对不平衡不敏感
- 精确率和召回率有权衡
- F1平衡两者
- AUC适合模型比较

**记住**：没有"最好"的指标，只有"最合适"的指标！
