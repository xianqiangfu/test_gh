# 随机森林

## 什么是随机森林？

随机森林是**多棵决策树的集成方法**。

简单说：**一群决策树投票决定最终结果。**

---

## 为什么叫"随机"？

两个方面的随机：

1. **样本随机**：每棵树用不同的训练样本
2. **特征随机**：每棵树考虑不同的特征

---

## 生活类比

### 专家投票

想象你要决定是否投资一个项目：

- 你问10个不同的专家
- 每个专家根据不同信息给出建议
- 最后多数投票决定

随机森林就是这样工作的！

---

## 随机森林的结构

### 组成

```
决策树1  ──→ 预测A ┐
                   ├── 投票 ──→ 最终预测
决策树2  ──→ 预测B ┤
                   │
决策树3  ──→ 预测A ┘
```

- 多棵决策树
- 每棵树独立训练
- 投票决定最终结果

---

## 工作原理

### 训练过程

1. **采样**
   - 用**Bootstrap采样**生成多个数据集
   - 每个数据集有重复和遗漏

2. **建树**
   - 每个数据集训练一棵决策树
   - 建树时只考虑部分特征

3. **集成**
   - 所有树组成森林

### 预测过程

1. **每棵树预测**
   - 每棵树独立给出预测

2. **投票（分类）**
   - 统计各类的票数
   - 票数最多的类获胜

3. **平均（回归）**
   - 所有树的预测值取平均

---

## Bootstrap 采样

### 什么是Bootstrap？

Bootstrap 是一种**有放回采样**的方法。

### 示例

原始数据：[1, 2, 3, 4, 5]

采样1：[2, 5, 2, 3, 1]（有重复，有遗漏）
采样2：[4, 1, 5, 4, 2]
采样3：[3, 3, 5, 1, 4]

每棵树用不同的采样结果训练。

### 为什么有放回？

- 让每棵树的训练集不同
- 增加模型的多样性

---

## 特征随机

### 随机选择特征

在树的每个节点分裂时，只考虑**部分特征**。

```
总特征数：100
每次分裂考虑：sqrt(100) = 10 个特征
```

### 为什么特征随机？

- 让每棵树更不同
- 避免都选相同的特征
- 提高模型多样性

---

## 随机森林 vs 单棵决策树

| 特点 | 决策树 | 随机森林 |
|------|--------|----------|
| 模型数量 | 1 | 多 |
| 过拟合 | 容易 | 不易 |
| 稳定性 | 低 | 高 |
| 解释性 | 强 | 弱 |
| 计算速度 | 快 | 慢（训练） |
| 预测速度 | 快 | 中等 |

---

## 为什么随机森林好？

### 1. 多样性

- 每棵树不同
- 错误不会重复

### 2. 投票机制

- 多数投票更准确
- 减少方差

### 3. Bagging 思想

- 平行训练多棵树
- 集成减少方差

### 4. 随机性

- 样本随机 + 特征随机
- 最大化多样性

---

## 优点

✅ **准确率高**
- 通常比单棵树好很多

✅ **不易过拟合**
- 随机性防止过拟合

✅ **稳定性好**
- 数据变化影响小

✅ **处理高维数据**
- 可以选择重要特征

✅ **并行训练**
- 每棵树可以并行

✅ **不需要数据预处理**
- 不需要标准化

---

## 缺点

❌ **不可解释**
- "黑盒"模型
- 难以解释决策过程

❌ **训练慢**
- 需要训练多棵树
- 占用内存多

❌ **预测速度中等**
- 需要所有树预测

❌ **调参复杂**
- 参数较多

---

## 重要参数

| 参数 | 说明 | 默认值 | 推荐值 |
|------|------|--------|--------|
| n_estimators | 树的数量 | 100 | 100-500 |
| max_depth | 最大深度 | None | 5-15 |
| min_samples_split | 分裂最小样本数 | 2 | 2-10 |
| min_samples_leaf | 叶子最小样本数 | 1 | 1-5 |
| max_features | 分裂考虑特征数 | sqrt | sqrt/log2 |
| bootstrap | 是否Bootstrap采样 | True | True |

### 调参建议

1. n_estimators：越多越好，但收益递减
2. max_depth：从5开始，逐步增加
3. max_features：sqrt（分类）或 1/3（回归）

---

## 代码示例

### Scikit-learn

```python
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score

# 加载数据
iris = load_iris()
X, y = iris.data, iris.target

# 分割数据
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# 创建随机森林
rf = RandomForestClassifier(
    n_estimators=100,
    max_depth=10,
    max_features='sqrt',
    n_jobs=-1  # 并行
)

# 训练
rf.fit(X_train, y_train)

# 预测
y_pred = rf.predict(X_test)

# 评估
print(accuracy_score(y_test, y_pred))

# 特征重要性
print(rf.feature_importances_)
```

### 回归问题

```python
from sklearn.ensemble import RandomForestRegressor

# 创建随机森林回归器
rf_reg = RandomForestRegressor(
    n_estimators=100,
    max_depth=10,
    max_features='auto',
    n_jobs=-1
)

# 训练
rf_reg.fit(X_train, y_train)

# 预测
y_pred = rf_reg.predict(X_test)
```

## OOB误差

### 什么是OOB？

Out-of-Bag（包外）数据是Bootstrap采样中没有用到的样本。

### OOB估计

- 每棵树训练后，用OOB数据评估
- 所有树的OOB误差平均
- 类似交叉验证

### 使用

```python
rf = RandomForestClassifier(oob_score=True)
rf.fit(X, y)

print(f"OOB准确率: {rf.oob_score_}")
```

## 常见问题

### Q: 树的数量越多越好吗？

A: 一般是的，但收益递减。100棵树通常是好的起点。

### Q: 随机森林会过拟合吗？

A: 不容易，但如果树太多、太深，也可能轻微过拟合。

### Q: 随机森林 vs 梯度提升？

A:
- 随机森林：并行训练，稳定性好
- 梯度提升：串行训练，准确率可能更高

## 总结

**随机森林**：

1. 多棵决策树的集成
2. 通过样本和特征的随机性增加多样性
3. 投票决定最终结果
4. 准确率高，不易过拟合
5. 是强大的基准模型

**核心思想**：
- Bootstrap采样
- 随机特征选择
- 多树投票

**关键要点**：
- n_estimators 控制树的数量
- max_depth 防止过拟合
- 可以计算特征重要性
- 可以用OOB估计

**记住**：随机森林就像"民主投票"，虽然每个人（树）可能犯小错，但多数人的决定通常是正确的！
