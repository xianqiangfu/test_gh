# 决策树

## 什么是决策树？

决策树是一种**树形结构的分类和回归算法**。

简单说：**通过一系列"是/否"的问题，一步步做出决策。**

---

## 生活类比

### 找朋友的例子

你在找人，问了一连串问题：

```
问：他/她是男生吗？
答：是 → 问：他戴眼镜吗？
           答：是 → 可能是张三
           答：否 → 可能是李四
答：否 → 问：她长发吗？
           答：是 → 可能是王五
           答：否 → 可能是赵六
```

这就是一棵决策树！

---

## 决策树的结构

### 组成部分

```
                    [是]
           ┌─────────┴─────────┐
        [戴眼镜?]            [长发?]
         │   │                │   │
      [是] [否]           [是] [否]
        │    │               │    │
      张三  李四          王五  赵六
```

- **根节点**：树的顶端，第一个问题
- **内部节点**：中间的问题
- **叶节点**：最终的决策结果
- **分支**：问题的答案路径

---

## 决策树的工作原理

### 建树过程

1. **选择最佳特征**
   - 找最能区分数据的特征

2. **设置分割点**
   - 确定如何分割数据

3. **递归建树**
   - 对每个子集重复上述过程
   - 直到满足停止条件

4. **生成决策**
   - 叶节点给出预测结果

### 停止条件

- 所有样本属于同一类别
- 没有可用特征了
- 达到最大深度
- 节点样本数太少

---

## 如何选择最佳特征？

### 信息增益

基于**熵**和**信息增益**。

**熵**：衡量数据的不确定性。

```
熵 = -Σ pᵢ × log₂(pᵢ)
```

**信息增益**：分割后熵的减少量。

```
信息增益 = 熵(分割前) - 熵(分割后)
```

选择信息增益最大的特征。

### 基尼指数

另一种衡量不纯度的方式。

```
基尼指数 = 1 - Σ pᵢ²
```

选择基尼指数减少最多的特征。

---

## 分类树 vs 回归树

### 分类树

**任务**：预测类别
**叶节点**：多数投票决定类别
**衡量标准**：信息增益、基尼指数

### 回归树

**任务**：预测数值
**叶节点**：平均值作为预测
**衡量标准**：均方误差 (MSE)

## 优点

✅ **直观易懂**
- 可以可视化
- 决策过程透明

✅ **不需要数据预处理**
- 不需要标准化
- 可以处理缺失值

✅ **处理非线性关系**
- 天然支持非线性

✅ **特征重要**
- 可以看出哪些特征重要

✅ **可解释性强**
- 每个决策都可以解释

---

## 缺点

❌ **容易过拟合**
- 树太深会记住噪声
- 需要剪枝

❌ **不稳定**
- 数据轻微变化，树可能大变
- 可以用随机森林解决

❌ **贪婪算法**
- 每次只找局部最优
- 不保证全局最优

❌ **偏向多值特征**
- 特征值多的容易被选中

---

## 剪枝

### 为什么要剪枝？

完整的决策树容易过拟合。剪枝可以简化树，提高泛化能力。

### 预剪枝（先剪）

在建树过程中停止：

- 限制最大深度
- 限制最小叶子样本数
- 限制最小分裂样本数

### 后剪枝（后剪）

先建完整树，再修剪：

- 计算剪枝前后的验证集误差
- 修剪不提高性能的分支

## 代码示例

### 分类树

```python
from sklearn.tree import DecisionTreeClassifier
from sklearn.datasets import load_iris

# 加载数据
iris = load_iris()
X, y = iris.data, iris.target

# 创建决策树
clf = DecisionTreeClassifier(max_depth=3)

# 训练
clf.fit(X, y)

# 预测
print(clf.predict([[5.1, 3.5, 1.4, 0.2]]))

# 特征重要性
print(clf.feature_importances_)
```

### 回归树

```python
from sklearn.tree import DecisionTreeRegressor

# 创建决策树
reg = DecisionTreeRegressor(max_depth=3)

reg.fit(X_train, y_train)

# 预测
```

## 参数调优

### 重要参数

| 参数 | 说明 | 默认值 | 推荐值 |
|------|------|--------|--------|
| max_depth | 最大深度 | None | 3-10 |
| min_samples_split | 分裂最小样本数 | 2 | 2-10 |
| min_samples_leaf | 叶子最小样本数 | 1 | 1-5 |
| max_features | 分裂考虑特征数 | None | sqrt/log2 |
| criterion | 分裂标准 | gini | gini/entropy |

### 调参策略

1. 先用默认值
2. 观察是否过拟合
3. 用交叉验证调参

## 常见问题

### Q: 决策树容易过拟合吗？

A: 是的，完整决策树容易过拟合。需要剪枝或限制深度。

### Q: 如何防止过拟合？

A:
- 限制最大深度
- 增加最小叶子样本数
- 使用交叉验证
- 用随机森林

### Q: 决策树 vs 随机森林？

A:
- 决策树：简单、可解释，但容易过拟合
- 随机森林：多棵树投票，更稳定

## 总结

**决策树**：

1. 通过一系列"是/否"问题做决策
2. 可视化、可解释性强
3. 不需要数据预处理
4. 容易过拟合，需要剪枝
5. 是集成学习的基础

**核心思想**：
- 选择最佳特征分割数据
- 递归构建树
- 叶节点给出预测

**关键要点**：
- 直观易懂，适合非专业人士
- 需要剪枝防止过拟合
- 是随机森林的基础
- 特征重要性分析有用

**记住**：决策树就像"20个问题"游戏，通过聪明的问题快速找到答案！
