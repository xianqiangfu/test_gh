# 梯度提升树

## 什么是梯度提升树？

梯度提升树（Gradient Boosting Trees，GBT）是**梯度下降在函数空间的应用**。

简单说：**逐步添加弱学习器，每次专注于修正之前的错误。**

---

## 核心思想

### 累加模型

```
预测结果 = 树1 + 树2 + 树3 + ... + 树N
```

### 每一步修正

```
树1：粗略预测
树2：修正树1的错误
树3：修正树1+树2的错误
...
树N：最终修正
```

---

## 生活类比

### 纠错游戏

想象你在画图：

1. 画第一版：很粗糙
2. 发现错误，画第二版：更好了
3. 还不够，画第三版：又改进了
4. ...
5. 最终版：已经很准确了

梯度提升就是这样一步步改进的！

---

## 工作原理

### 步骤

1. **初始化**
   - 用简单模型（如均值）初始化

2. **计算残差**
   - 计算预测值和真实值的差距

3. **拟合残差**
   - 新模型预测残差

4. **更新模型**
   - 把新模型加到旧模型上

5. **重复**
   - 直到达到指定树的数量

### 数学形式

```
F₀(x) = 初始预测
Fₘ(x) = Fₘ₋₁(x) + 学习率 × 树ₘ(x)
```

---

## 梯度提升 vs 随机森林

| 特点 | 随机森林 | 梯度提升 |
|------|----------|----------|
| 训练方式 | 并行 | 串行 |
| 每棵树 | 独立 | 修正前一棵树的错误 |
| 重点 | 减少方差 | 减少偏差和方差 |
| 过拟合 | 不易 | 较易 |
| 训练速度 | 快 | 慢 |
| 准确率 | 高 | 很高 |

---

## 学习率

### 什么是学习率？

控制每棵树的贡献大小。

```
Fₘ(x) = Fₘ₋₁(x) + 学习率 × 树ₘ(x)
```

### 学习率的影响

- 学习率大：收敛快，但可能过拟合
- 学习率小：收敛慢，但更稳定

### 和树数量的关系

```
学习率小 → 需要更多树
学习率大 → 需要更少树
```

通常：学习率 × 树数量 ≈ 常数

---

## 正则化方法

### 1. 限制树深度

```
max_depth: 3-6
```

浅树可以防止过拟合。

### 2. 子采样

每棵树用部分数据训练：

```
subsample: 0.5-0.8
```

### 3. 特征子采样

每次分裂用部分特征：

```
max_features: sqrt 或 log2
```

### 4. 学习率

小学习率 + 多树：

```
learning_rate: 0.01-0.1
n_estimators: 100-1000
```

## 代码示例

### XGBoost

```python
import xgboost as xgb
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score

# 加载数据
data = load_breast_cancer()
X, y = data.data, data.target

# 分割数据
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# 创建模型
model = xgb.XGBClassifier(
    n_estimators=100,
    max_depth=3,
    learning_rate=0.1,
    subsample=0.8,
    colsample_bytree=0.8
)

# 训练
model.fit(X_train, y_train)

# 预测
y_pred = model.predict(X_test)

# 评估
print(accuracy_score(y_test, y_pred))

# 特征重要性
model.plot_importance(model)
```

### LightGBM

```python
import lightgbm as lgb

# 创建模型
model = lgb.LGBMClassifier(
    n_estimators=100,
    max_depth=3,
    learning_rate=0.1,
    subsample=0.8
)

# 训练
model.fit(X_train, y_train)

# 预测
y_pred = model.predict(X_test)
```

### CatBoost

```python
import catboost as cb

# 创建模型
model = cb.CatBoostClassifier(
    iterations=100,
    depth=3,
    learning_rate=0.1
)

# 训练
model.fit(X_train, y_train)

# 预测
y_pred = model.predict(X_test)
```

## 参数调优

### 重要参数

| 参数 | 说明 | 默认值 | 推荐值 |
|------|------|--------|--------|
| n_estimators | 树的数量 | 100 | 100-1000 |
| max_depth | 最大深度 | 3-6 | 3-6 |
| learning_rate | 学习率 | 0.1 | 0.01-0.2 |
| subsample | 样本子采样 | 1.0 | 0.6-0.8 |
| colsample_bytree | 特征子采样 | 1.0 | 0.6-0.8 |

### 调参策略

1. 固定 max_depth=3
2. 调整 n_estimators 和 learning_rate
3. 用子采样防止过拟合
4. 用交叉验证评估

## 常见问题

### Q: 梯度提升 vs 随机森林？

A:
- 梯度提升：准确率更高，训练更慢
- 随机森林：更快更稳定，准确率略低

### Q: 如何防止过拟合？

A:
- 减小学习率
- 限制树深度
- 使用子采样
- 早停

### Q: 学习率怎么选？

A:
- 0.01-0.1 是常见范围
- 学习率小用更多树
- 学习率大用更少树

## 总结

**梯度提升树**：

1. 逐步添加弱学习器
2. 每次修正之前的错误
3. 学习率控制贡献大小
4. 准确率高但训练慢
5. 是很多竞赛获胜的方法

**核心思想**：
- 梯度下降在函数空间
- 逐步优化
- 减少偏差和方差

**关键要点**：
- 学习率 × 树数量 ≈ 常数
- 用正则化防止过拟合
- XGBoost/LightGBM 是高效实现
- 适合追求高准确率的场景

**记住**：梯度提升就像"精细打磨"，一步步改进，最终达到很高的精度！
