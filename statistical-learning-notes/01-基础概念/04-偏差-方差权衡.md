# 偏差-方差权衡

## 问题的起源

在机器学习中，我们经常会遇到一个矛盾：

- 想让模型拟合数据更准确 → 模型变复杂，可能过拟合
- 想让模型更稳定可靠 → 模型变简单，可能欠拟合

这就引出了**偏差-方差权衡**这个核心概念。

---

## 什么是误差？

模型的误差主要来自三个部分：

```
总误差 = 偏差² + 方差 + 不可避免的噪声
```

### 1. 偏差 (Bias)

**偏差**是模型**预测值和真实值之间的平均差异**。

**简单理解**：偏差反映了模型**不够聪明**的程度。

**高偏差的表现**：
- 模型太简单，无法捕捉数据的真实规律
- 训练集和测试集的误差都很大
- 欠拟合（Underfitting）

**例子**：
- 用一条直线去拟合弯弯曲曲的数据
- 无论怎么训练，都拟合不好

### 2. 方差 (Variance)

**方差**是模型**对数据变化的敏感程度**。

**简单理解**：方差反映了模型**不够稳定**的程度。

**高方差的表现**：
- 模型太复杂，对数据变化太敏感
- 训练集误差很小，但测试集误差很大
- 过拟合（Overfitting）

**例子**：
- 用一条极其复杂的曲线去拟合少量数据
- 训练集完美拟合，但新数据就崩溃了

### 3. 不可避免的噪声

这是数据本身的问题，无法通过模型来解决。

## 偏差-方差权衡的曲线

```
误差
  ↑
  │    ╲ 总误差
  │     ╲
  │      ╲
  │       ╲
  │        ╲
  │    ────╲────────────── 方差
  │    ╱    ╲
  │   ╱      ╲
  │  ╱        ╲──── 偏差
  │ ╱
  │╱
  └────────────────────→ 模型复杂度
    简单        适中        复杂
```

**关键点**：
- 简单模型：高偏差，低方差
- 复杂模型：低偏差，高方差
- 最优点：总误差最小的位置

---

## 如何平衡？

### 1. 从数据量考虑

**数据少** → 用简单模型（降低方差风险）
**数据多** → 可以用复杂模型（降低偏差）

### 2. 从问题复杂度考虑

**简单问题** → 用简单模型即可
**复杂问题** → 需要复杂模型

### 3. 使用正则化

正则化可以在控制模型复杂度的同时，平衡偏差和方差。

### 4. 交叉验证

用交叉验证来找到模型复杂度的最佳平衡点。

---

## 实际例子

### 例子1：房价预测

**方案A**（高偏差）：
- 只用"房屋面积"一个特征
- 用简单线性回归
- 结果：预测太粗糙，偏差大

**方案B**（高方差）：
- 用几百个特征（包括噪音）
- 用深度神经网络
- 结果：训练集完美，测试集很差

**方案C**（平衡）：
- 用十几个重要特征
- 用中等复杂度的模型
- 结果：训练和测试都不错

---

## 诊断方法

### 如何判断是高偏差还是高方差？

| 现象 | 问题 | 解决方案 |
|------|------|----------|
| 训练误差很大，测试误差也很大 | 高偏差 | 用更复杂的模型、增加特征 |
| 训练误差很小，测试误差很大 | 高方差 | 简化模型、增加数据、正则化 |

---

## 总结

**偏差-方差权衡**的核心：

1. **偏差** = 模型太简单，学不到规律
2. **方差** = 模型太复杂，学了太多噪声
3. **目标** = 找到平衡点，总误差最小

**实用建议**：
- 先检查是高偏差还是高方差
- 数据少时偏向简单模型
- 数据多时可以尝试复杂模型
- 用交叉验证找最佳点

记住：**完美的模型不存在，但我们可以找到最优的平衡。**
