# 软间隔

## 什么是软间隔？

软间隔是**对硬间隔SVM的扩展**，允许一些分类错误。

简单说：**不再要求完美分类，允许一些样本被分错。**

---

## 为什么需要软间隔？

### 硬间隔的问题

1. **数据不是线性可分**
   ```
      ●●●●●●
           ●
   ●●●●    ○○○○
       ○○○○○○
   ```
   没有直线能完全分开这两类。

2. **对异常值敏感**
   ```
   ●●●●●●
       ●← 异常点
   ●●●●━━━━○○○○
       ○○○○○○
   ```
   一个异常点就破坏了完美的分类。

### 软间隔的优势

- 可以处理线性不可分数据
- 对异常值更鲁棒
- 更实用

---

## 软间隔的思想

### 允许错误

不再要求所有样本都正确分类，允许一些错误：

```
    ●●●●●●
         ●
●●●●     ○○○○
     ○○○○○○
```

虽然有一个点被分错了，但整体更好。

### 权衡

- 错误越少越好
- 但不能为了少数点牺牲太多

---

## 数学表示

### 松弛变量

引入松弛变量 ξᵢ：

```
yᵢ(wᵀxᵢ + b) ≥ 1 - ξᵢ
ξᵢ ≥ 0
```

- ξᵢ = 0：正确分类，且在间隔边界外
- 0 < ξᵢ < 1：正确分类，但在间隔内
- ξᵢ ≥ 1：被错误分类

### 优化问题

```
最小化：1/2 × ||w||² + C × Σξᵢ
约束条件：yᵢ(wᵀxᵢ + b) ≥ 1 - ξᵢ，ξᵢ ≥ 0
```

- C：惩罚参数
- ξᵢ：松弛变量

---

## 参数 C 的影响

### C 的作用

C 控制对错误的惩罚程度：

```
C 大 → 对错误惩罚重 → 接近硬间隔
C 小 → 对错误惩罚轻 → 更宽松
```

### C 的选择

```
C 很大：
- 几乎不允许错误
- 接近硬间隔
- 可能过拟合

C 适中：
- 允许一些错误
- 平衡间隔和错误
- 通常最好

C 很小：
- 允许很多错误
- 间隔可能很大
- 可能欠拟合
```


## 损失函数

### Hinge 损失

软间隔SVM使用Hinge损失：

```
L(y, f(x)) = max(0, 1 - y × f(x))
```

- 如果 y×f(x) ≥ 1，损失为0
- 如果 y×f(x) < 1，损失为 1 - y×f(x)

### 损失曲线

```
损失
  ↑
  │  ╱│
  │ ╱ │
  │╱  │
  ├───┴────→ y×f(x)
  │0   1
```

- 正确分类且在间隔外：损失0
- 否则：线性增加

---

## 对偶问题

### 软间隔的对偶

```
最大化：Σαᵢ - 1/2 ΣΣαᵢαⱼyᵢyⱼK(xᵢ, xⱼ)
约束条件：0 ≤ αᵢ ≤ C，Σαᵢyᵢ = 0
```

- αᵢ ≤ C：上界约束
- 其他与硬间隔相同

---

## 支持向量的类型

### 边界支持向量

```
0 < αᵢ < C
```

正好在间隔边界上的点。

### 错误支持向量

```
αᵢ = C
```

被错误分类或在间隔内的点。

### 非支持向量

```
αᵢ = 0
```

不影响决策边界的点。

## 参数调优

### C 的调参

用网格搜索找最佳 C：

```python
from sklearn.svm import SVC
from sklearn.model_selection import GridSearchCV

param_grid = {'C': [0.01, 0.1, 1, 10, 100]}

grid = GridSearchCV(SVC(kernel='rbf'), param_grid, cv=5)
grid.fit(X, y)

print(grid.best_params_)
```

### C 的推荐范围

- 从 [0.1, 1, 10, 100] 开始
- 根据结果调整

## 代码示例

### Scikit-learn

```python
from sklearn.svm import SVC
from sklearn.datasets import make_classification
import matplotlib.pyplot as plt

# 生成数据
X, y = make_classification(n_samples=100, n_features=2, n_redundant=0,
                           n_informative=2, random_state=42, n_clusters_per_class=1)

# 创建模型（软间隔）
clf = SVC(kernel='linear', C=1.0)  # C=1 软间隔

# 训练
clf.fit(X, y)

# 查看支持向量
print(f"支持向量数量: {len(clf.support_vectors_)}")

# 绘制决策边界
def plot_decision_boundary(clf, X, y):
    x_min, x_max = X[:, 0].min() - 1, X[:, 0].max() + 1
    y_min, y_max = X[:, 1].min() - 1, X[:, 1].max() + 1
    xx, yy = np.meshgrid(np.arange(x_min, x_max, 0.02),
                         np.arange(y_min, y_max, 0.02))
    Z = clf.predict(np.c_[xx.ravel(), yy.ravel()])
    Z = Z.reshape(xx.shape)

    plt.contourf(xx, yy, Z, alpha=0.3)
    plt.scatter(X[:, 0], X[:, 1], c=y)
    plt.scatter(clf.support_vectors_[:, 0], clf.support_vectors_[:, 1],
                s=100, facecolors='none', edgecolors='r', label='支持向量')
    plt.legend()
    plt.show()

plot_decision_boundary(clf, X, y)
```

## 与硬间隔比较

| 特点 | 硬间隔 | 软间隔 |
|------|--------|--------|
| 错误 | 不允许 | 允许 |
| 适用性 | 线性可分 | 更通用 |
| 鲁棒性 | 对噪声敏感 | 更鲁棒 |
| 参数 | 无 | 需要调 C |
| 实用性 | 理论价值 | 实用价值 |

## 实际应用

### 数据有噪声

实际数据总有噪声，软间隔更实用。

### 异常值处理

软间隔可以容忍一些异常值。

### 非线性可分

结合核函数，处理复杂问题。

---

## 常见问题

### Q: C 越大越好吗？

A: 不是。C 太大可能过拟合，C 太小可能欠拟合。

### Q: 如何判断软间隔是否合适？

A:
- 用交叉验证评估
- 比较不同 C 的表现
- 观察训练集和测试集的差距

### Q: 软间隔 + 核函数？

A: 可以，这是最常用的SVM配置。

---

## 总结

**软间隔**：

1. 允许一些分类错误
2. 引入松弛变量 ξᵢ
3. 参数 C 控制惩罚程度
4. 比硬间隔更实用
5. 通常与核函数一起使用

**核心思想**：
- 权衡间隔和错误
- 允许一些错误
- 更鲁棒更实用

**关键要点**：
- C 大：接近硬间隔
- C 小：更宽松
- 用交叉验证调 C
- 支持向量有不同类型

**记住**：软间隔就像"宽容的老师"，允许学生犯一些小错，只要整体表现好就行！
