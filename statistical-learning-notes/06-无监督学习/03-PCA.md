# 主成分分析

## 什么是PCA？

PCA（Principal Component Analysis）是最重要的**降维技术**之一。

简单说：**找到数据最重要的方向，用更少的维度表示数据，同时保留最多信息。**

---

## 为什么需要降维？

### 维度灾难

数据维度越高：

- 计算成本高
- 数据稀疏
- 模型复杂
- 难以可视化

### 降维的好处

- 减少存储和计算
- 可视化高维数据
- 去除噪声
- 提高模型性能

---

## 核心思想

### 变换

把数据从原始坐标系变换到新的坐标系：

```
原始坐标系       新坐标系
y↑             PC2↑
 │             │
 │             │
 └────→x       └────→PC1
```

- PC1：第一主成分（最重要）
- PC2：第二主成分（次重要）
- ...

### 目标

- 新维度之间不相关
- 第一个主成分保留最多方差
- 后续主成分依次保留最多剩余方差

---

## 数学原理

### 方差

衡量数据的离散程度：

```
方差 = Σ(xᵢ - x̄)² / n
```

方差越大，信息越多。

### 协方差

衡量两个变量的关系：

```
协方差(x, y) = Σ(xᵢ - x̄)(yᵢ - ȳ) / n
```

- 协方差 > 0：正相关
- 协方差 < 0：负相关
- 协方差 = 0：不相关

### 协方差矩阵

```
Σ = [ var(x)      cov(x,y)
      cov(y,x)    var(y)   ]
```

---

## PCA的步骤

### 步骤1：标准化

把每个特征标准化到0均值、1方差：

```
x' = (x - 均值) / 标准差
```

### 步骤2：计算协方差矩阵

```
Σ = XᵀX / (n-1)
```

### 步骤3：特征分解

```
Σ v = λ v
```

- v：特征向量（主成分方向）
- λ：特征值（方差大小）

### 步骤4：选择主成分

按特征值从大到小排序：

```
λ₁ ≥ λ₂ ≥ ... ≥ λₚ
```

选择前 k 个主成分。

### 步骤5：转换数据

```
X' = X × Vₖ
```

Vₖ 是前 k 个特征向量。

```

```

```
  │   ●  ●
  │  ●   ●

  去除相关性
```

### 降维

```
PC2↑
  │   ●  ●
  │  ●   ●
  │ ●     ●
  └────────→PC1
  只保留PC1
```

## 解释方差

### 方差解释比

第 i 个主成分解释的方差比例：

```
解释比ᵢ = λᵢ / Σλⱼ
```

### 累积方差解释比

前 k 个主成分的累积解释比：

```
累积比ₖ = Σλᵢ / Σλⱼ   (i=1..k)
```

### 例子

```
主成分   特征值   解释比   累积比
PC1      50      50%      50%
PC2      30      30%      80%
PC3      10      10%      90%
PC4      5       5%       95%
PC5      5       5%       100%
```

前2个主成分解释了80%的方差。

---

## 选择主成分数量

### 方法1：累积方差

选择解释85%-95%方差的主成分：

```
累积比 ≥ 0.85
```

### 方法2：肘部法则

像K-Means一样找肘部点：

```
特征值
  │
  │      ╱╲
  │     ╱  ╲
  │    ╱    ╲
  │   ╱      ╲
  │  ╱        ╲
  └──────────────→ 主成分
       肘部
```

### 方法3：Kaiser准则

保留特征值 > 1 的主成分。

### 方法4：实际需求

根据实际应用需求选择。

## 代码示例

### Scikit-learn

```python
from sklearn.decomposition import PCA
from sklearn.datasets import load_iris
import matplotlib.pyplot as plt

# 加载数据
iris = load_iris()
X = iris.data

# PCA
pca = PCA()
X_pca = pca.fit_transform(X)

# 查看解释方差
print(f"解释方差比: {pca.explained_variance_ratio_}")
print(f"累积解释方差: {pca.explained_variance_ratio_.cumsum()}")

# 选择前2个主成分
pca_2 = PCA(n_components=2)
X_2d = pca_2.fit_transform(X)

# 可视化
plt.scatter(X_2d[:, 0], X_2d[:, 1], c=iris.target)
plt.xlabel('PC1')
plt.ylabel('PC2')
plt.title('PCA降维结果')
plt.colorbar()
plt.show()
```

---

## 优点

✅ **去除相关性**
- 主成分之间不相关

✅ **降维**
- 减少维度和计算量

✅ **可解释**
- 每个主成分的贡献明确

✅ **理论基础**
- 完善的数学理论

---

## 缺点

❌ **线性变换**
- 只能处理线性关系

❌ **可解释性**
- 主成分含义不直观

❌ **对异常值敏感**
- 需要处理异常值

❌ **假设正态分布**
- 效果依赖于分布

---

## 常见问题

### Q: PCA 会丢失信息吗？

A:
- 会，但丢失的是次要信息
- 主成分保留最多方差

### Q: 如何选择主成分数量？

A:
- 累积方差 ≥ 85%
- 肘部法则
- 实际需求

### Q: PCA 之前需要标准化吗？

A:
- 需要！
- 不同尺度的特征会影响结果

---

## 总结

**PCA（主成分分析）**：

1. 找到数据最重要的方向
2. 用更少维度保留最多信息
3. 主成分之间不相关
4. 基于方差最大化
5. 降维、去噪、可视化

**核心思想**：
- 找方差最大的方向
- 主成分之间正交
- 降维保留信息

**关键要点**：
- 标准化很重要
- 累积方差决定主成分数
- 线性变换，只处理线性关系
- 可解释性好

**记住**：PCA就像"找重点"，把复杂的东西简化，保留最重要的部分！
