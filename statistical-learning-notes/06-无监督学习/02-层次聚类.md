# 层次聚类

## 什么是层次聚类？

层次聚类是**构建树形结构的聚类算法**。

简单说：**逐步合并或分割簇，形成层级结构。**

---

## 核心思想

### 层次结构

```
        ┌────────────────┐
        │     所有数据    │
        └────────┬───────┘
                 │
        ┌────────┴────────┐
        │                 │
    ┌───┴───┐         ┌───┴───┐
    │ 簇1  │         │ 簇2  │
    └───┬───┘         └───┬───┘
        │                 │
    ┌───┴───┐         ┌───┴───┐
    │子簇1.1│         │子簇2.1│
    └───────┘         └───────┘
```

形成一个树形结构（树状图）。

---

## 两种类型

### 自底向上（凝聚式）

从每个点一个簇开始，逐步合并：

```
●  ●  ●  ●  ●  ●
│  │  │  │  │  │
└──┴──┴──┴──┴──┴─── 逐步合并
```

### 自顶向下（分裂式）

从所有点一个簇开始，逐步分割：

```
●  ●  ●  ●  ●  ●
│  │  │  │  │  │
└──┴──┴──┴──┴──┴─── 逐步分割
```

最常用的是凝聚式。

---

## 凝聚式层次聚类

### 算法步骤

1. **初始化**
   - 每个点是一个簇

2. **计算距离**
   - 计算所有簇之间的距离

3. **合并最近簇**
   - 合并距离最近的两个簇

4. **更新距离**
   - 计算新簇与其他簇的距离

5. **重复**
   - 直到只剩一个簇

### 示例

```
初始：A, B, C, D, E
      ↓
合并A和B：{AB}, C, D, E
      ↓
合并{AB}和C：{ABC}, D, E
      ↓
合并D和E：{ABC}, {DE}
      ↓
合并{ABC}和{DE}：{ABCDE}
```

---

## 距离度量

### 点之间的距离

- 欧几里得距离
- 曼哈顿距离
- 余弦距离

### 簇之间的距离（链接方式）

#### 1. 单链接（最小距离）

```
d(C₁, C₂) = min{d(xᵢ, xⱼ) | xᵢ∈C₁, xⱼ∈C₂}
```

取两个簇中最近的两个点的距离。

**特点**：
- 可能产生"链式"效果
- 能处理非球形簇

#### 2. 完全链接（最大距离）

```
d(C₁, C₂) = max{d(xᵢ, xⱼ) | xᵢ∈C₁, xⱼ∈C₂}
```

取两个簇中最远的两个点的距离。

**特点**：
- 倾向于产生紧凑簇
- 对异常值敏感

#### 3. 平均链接

```
d(C₁, C₂) = average{d(xᵢ, xⱼ) | xᵢ∈C₁, xⱼ∈C₂}
```

取两个簇中所有点对的平均距离。

**特点**：
- 折中单链接和完全链接
- 通常效果较好

#### 4. Ward 方法

最小化合并后的方差增量。

**特点**：
- 倾向于产生大小相近的簇
- 通常效果好

## 树状图

### 什么是树状图？

树状图是层次聚类的可视化：

```
距离
  │
  │      ┌─────────┬─────────┐
  │    ──┤         ├─────────┤
  │  ──┤   ├───────┤
  │──┤   ├───┤     │
  │  │   │   │     │
  A  B   C   D     E
```

### 如何读取？

- 纵轴：合并距离
- 横轴：样本点
- 合并的位置：高度表示簇之间的距离

### 切割位置

决定分成几簇：

```
距离
  │
  │      ┌─────────┬─────────┐
  │    ──┤         ├─────────┤
  │  ──┤   ├───────┤
  │──┤   ├───┤     │
  ──┤─────┤─────────┤ ← 切割在这里
  │  │   │   │     │
  A  B   C   D     E

结果：3个簇
```

## 代码示例

### Scikit-learn

```python
from sklearn.cluster import AgglomerativeClustering
from scipy.cluster.hierarchy import dendrogram, linkage
import matplotlib.pyplot as plt

# 生成数据
X, y = make_blobs(n_samples=100, centers=3, random_state=42)

# 层次聚类
agg = AgglomerativeClustering(n_clusters=3)
labels = agg.fit_predict(X)

# 绘制聚类结果
plt.scatter(X[:, 0], X[:, 1], c=labels)
plt.title('层次聚类结果')
plt.show()

# 绘制树状图
Z = linkage(X, 'ward')
dendrogram(Z)
plt.title('树状图')
plt.xlabel('样本')
plt.ylabel('距离')
plt.show()
```

## 优点

✅ **不需要指定簇数**
- 可以通过树状图选择

✅ **层次结构**
- 可以得到不同粒度的聚类

✅ **可视化**
- 树状图直观

✅ **灵活性**
- 可以选择不同的链接方式

---

## 缺点

❌ **计算复杂度高**
- O(n³) 或 O(n²)
- 不适合大数据

❌ **不可逆**
- 一旦合并，不能撤销

❌ **对异常值敏感**
- 尤其是完全链接

❌ **难以选择链接方式**
- 不同方法结果不同

---

## 与K-Means比较

| 特点 | K-Means | 层次聚类 |
|------|---------|----------|
| 需要指定K | 是 | 否 |
| 层次结构 | 无 | 有 |
| 计算复杂度 | O(nkt) | O(n²) 或 O(n³) |
| 大数据 | 适合 | 不适合 |
| 初始敏感 | 是 | 否 |
| 灵活性 | 较低 | 较高 |

## 链接方式选择

### 单链接

- 适合：细长、非球形簇
- 不适合：噪声大

### 完全链接

- 适合：紧凑的球形簇
- 不适合：链式数据

### 平均链接

- 通常：效果较好
- 适合：一般情况

### Ward

- 通常：效果最好
- 适合：大多数情况

---

## 常见问题

### Q: 如何选择链接方式？

A:
- 默认Ward方法
- 有先验知识时用对应方式
- 用树状图观察结果

### Q: 层次聚类适合大数据吗？

A:
- 不适合，计算复杂度太高
- 大数据用K-Means或Mini-Batch K-Means

### Q: 如何确定簇数？

A:
- 观察树状图
- 选择"肘部"位置
- 结合业务知识

## 总结

**层次聚类**：

1. 构建树形结构
2. 自底向上逐步合并
3. 不需要指定簇数
4. 可以可视化
5. 计算复杂度高

**核心思想**：
- 从小到大逐步合并
- 形成层级结构
- 树状图展示

**关键要点**：
- 不同链接方式效果不同
- Ward方法通常最好
- 不适合大数据
- 树状图帮助选择簇数

**记住**：层次聚类就像"组织架构"，从个体到部门，从部门到公司，层层递进！
